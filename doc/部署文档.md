# 部署文档

## 1. 环境准备

### 1.1 开发环境

#### 必需软件
- Go 1.22+
- Docker 20.10+
- Docker Compose 2.0+
- Make
- Git

#### 可选软件
- PostgreSQL 15+ (本地调试)
- Redis 7+ (本地调试)
- goctl (go-zero 工具)

### 1.2 生产环境

#### AWS 服务
- ECS/EKS: 容器编排
- Aurora PostgreSQL: 数据库
- ElastiCache Redis: 缓存
- MSK/SQS: 消息队列
- S3: 对象存储
- CloudFront: CDN
- ALB: 负载均衡
- Route53: DNS
- CloudWatch: 监控
- Systems Manager: 配置管理

## 2. 本地开发部署

### 2.1 克隆项目

```bash
git clone https://github.com/jacklau/audio-ai-platform.git
cd audio-ai-platform
```

### 2.2 安装依赖

```bash
# 安装 Go 依赖
make deps

# 安装 goctl
make goctl-install
```

### 2.3 启动基础服务

```bash
# 启动 PostgreSQL, Redis, Kafka, LocalStack
make docker-up

# 等待服务就绪
sleep 10

# 运行数据库迁移
make migrate-up
```

### 2.4 配置服务

编辑各服务的配置文件：

#### 用户服务配置
```yaml
# services/user/etc/user.yaml
Name: user-api
Host: 0.0.0.0
Port: 8001

DB:
  DataSource: host=localhost port=5432 user=admin password=admin123 dbname=audio_platform sslmode=disable

Redis:
  Host: localhost:6379
  Pass: redis123
  Type: node

Auth:
  AccessSecret: your-secret-key
  AccessExpire: 3600
```

### 2.5 启动服务

```bash
# 方式 1: 使用 Make 启动所有服务
make dev

# 方式 2: 单独启动各服务
cd services/user && go run user.go -f etc/user.yaml
cd services/device && go run device.go -f etc/device.yaml
cd services/content && go run content.go -f etc/content.yaml
```

### 2.6 验证服务

```bash
# 检查用户服务
curl http://localhost:8001/health

# 检查设备服务
curl http://localhost:8002/health

# 检查内容服务
curl http://localhost:8003/health
```

## 3. Docker 部署

### 3.1 构建镜像

```bash
# 构建所有服务镜像
docker-compose build

# 或单独构建
docker build -t user-service:latest -f deploy/docker/user.Dockerfile .
docker build -t device-service:latest -f deploy/docker/device.Dockerfile .
docker build -t content-service:latest -f deploy/docker/content.Dockerfile .
```

### 3.2 启动容器

```bash
# 启动所有服务
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

### 3.3 Docker Compose 配置

```yaml
version: '3.8'

services:
  user-service:
    build:
      context: .
      dockerfile: deploy/docker/user.Dockerfile
    ports:
      - "8001:8001"
    environment:
      - DB_HOST=postgres
      - REDIS_HOST=redis
    depends_on:
      - postgres
      - redis

  device-service:
    build:
      context: .
      dockerfile: deploy/docker/device.Dockerfile
    ports:
      - "8002:8002"
    environment:
      - DB_HOST=postgres
      - REDIS_HOST=redis
    depends_on:
      - postgres
      - redis

  content-service:
    build:
      context: .
      dockerfile: deploy/docker/content.Dockerfile
    ports:
      - "8003:8003"
    environment:
      - DB_HOST=postgres
      - REDIS_HOST=redis
      - S3_ENDPOINT=http://localstack:4566
    depends_on:
      - postgres
      - redis
      - localstack
```

## 4. AWS ECS 部署

### 4.1 准备工作

#### 创建 ECR 仓库

```bash
aws ecr create-repository --repository-name audio-ai-platform/user-service
aws ecr create-repository --repository-name audio-ai-platform/device-service
aws ecr create-repository --repository-name audio-ai-platform/content-service
```

#### 推送镜像到 ECR

```bash
# 登录 ECR
aws ecr get-login-password --region us-east-1 | docker login --username AWS --password-stdin <account-id>.dkr.ecr.us-east-1.amazonaws.com

# 标记镜像
docker tag user-service:latest <account-id>.dkr.ecr.us-east-1.amazonaws.com/audio-ai-platform/user-service:latest

# 推送镜像
docker push <account-id>.dkr.ecr.us-east-1.amazonaws.com/audio-ai-platform/user-service:latest
```

### 4.2 创建 ECS 集群

```bash
aws ecs create-cluster --cluster-name audio-ai-platform-cluster
```

### 4.3 创建任务定义

```json
{
  "family": "user-service",
  "networkMode": "awsvpc",
  "requiresCompatibilities": ["FARGATE"],
  "cpu": "256",
  "memory": "512",
  "containerDefinitions": [
    {
      "name": "user-service",
      "image": "<account-id>.dkr.ecr.us-east-1.amazonaws.com/audio-ai-platform/user-service:latest",
      "portMappings": [
        {
          "containerPort": 8001,
          "protocol": "tcp"
        }
      ],
      "environment": [
        {
          "name": "DB_HOST",
          "value": "aurora-cluster.cluster-xxx.us-east-1.rds.amazonaws.com"
        },
        {
          "name": "REDIS_HOST",
          "value": "redis-cluster.xxx.cache.amazonaws.com"
        }
      ],
      "logConfiguration": {
        "logDriver": "awslogs",
        "options": {
          "awslogs-group": "/ecs/user-service",
          "awslogs-region": "us-east-1",
          "awslogs-stream-prefix": "ecs"
        }
      }
    }
  ]
}
```

### 4.4 创建服务

```bash
aws ecs create-service \
  --cluster audio-ai-platform-cluster \
  --service-name user-service \
  --task-definition user-service:1 \
  --desired-count 2 \
  --launch-type FARGATE \
  --network-configuration "awsvpcConfiguration={subnets=[subnet-xxx],securityGroups=[sg-xxx],assignPublicIp=ENABLED}" \
  --load-balancers "targetGroupArn=arn:aws:elasticloadbalancing:us-east-1:xxx:targetgroup/user-service/xxx,containerName=user-service,containerPort=8001"
```

## 5. Kubernetes 部署

### 5.1 创建 Deployment

```yaml
# deploy/k8s/user-service-deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: user-service
  namespace: audio-platform
spec:
  replicas: 3
  selector:
    matchLabels:
      app: user-service
  template:
    metadata:
      labels:
        app: user-service
    spec:
      containers:
      - name: user-service
        image: <account-id>.dkr.ecr.us-east-1.amazonaws.com/audio-ai-platform/user-service:latest
        ports:
        - containerPort: 8001
        env:
        - name: DB_HOST
          valueFrom:
            configMapKeyRef:
              name: app-config
              key: db_host
        - name: REDIS_HOST
          valueFrom:
            configMapKeyRef:
              name: app-config
              key: redis_host
        resources:
          requests:
            memory: "256Mi"
            cpu: "250m"
          limits:
            memory: "512Mi"
            cpu: "500m"
        livenessProbe:
          httpGet:
            path: /health
            port: 8001
          initialDelaySeconds: 30
          periodSeconds: 10
        readinessProbe:
          httpGet:
            path: /health
            port: 8001
          initialDelaySeconds: 5
          periodSeconds: 5
```

### 5.2 创建 Service

```yaml
# deploy/k8s/user-service-service.yaml
apiVersion: v1
kind: Service
metadata:
  name: user-service
  namespace: audio-platform
spec:
  selector:
    app: user-service
  ports:
  - protocol: TCP
    port: 80
    targetPort: 8001
  type: LoadBalancer
```

### 5.3 创建 ConfigMap

```yaml
# deploy/k8s/configmap.yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: app-config
  namespace: audio-platform
data:
  db_host: "aurora-cluster.cluster-xxx.us-east-1.rds.amazonaws.com"
  redis_host: "redis-cluster.xxx.cache.amazonaws.com:6379"
  aws_region: "us-east-1"
```

### 5.4 部署到 K8s

```bash
# 创建命名空间
kubectl create namespace audio-platform

# 应用配置
kubectl apply -f deploy/k8s/configmap.yaml
kubectl apply -f deploy/k8s/user-service-deployment.yaml
kubectl apply -f deploy/k8s/user-service-service.yaml

# 查看部署状态
kubectl get pods -n audio-platform
kubectl get services -n audio-platform

# 查看日志
kubectl logs -f deployment/user-service -n audio-platform
```

```

ECR_REPOSITORY: audio-ai-platform
```

## 8. 监控和日志

### 8.1 CloudWatch 日志

```bash
# 查看日志
aws logs tail /ecs/user-service --follow

# 查询日志
aws logs filter-log-events \
  --log-group-name /ecs/user-service \
  --filter-pattern "ERROR"
```

### 8.2 CloudWatch 指标

配置自定义指标：
- API 请求数
- API 响应时间
- 错误率
- 数据库连接数
- Redis 命中率

### 8.3 告警配置

```bash
# 创建告警
aws cloudwatch put-metric-alarm \
  --alarm-name user-service-high-error-rate \
  --alarm-description "User service error rate > 5%" \
  --metric-name ErrorRate \
  --namespace AudioPlatform \
  --statistic Average \
  --period 300 \
  --threshold 5 \
  --comparison-operator GreaterThanThreshold \
  --evaluation-periods 2
```

## 9. 扩容和缩容

### 9.1 手动扩容

```bash
# ECS 扩容
aws ecs update-service \
  --cluster audio-ai-platform-cluster \
  --service user-service \
  --desired-count 5

# K8s 扩容
kubectl scale deployment user-service --replicas=5 -n audio-platform
```

### 9.2 自动扩容

#### ECS Auto Scaling

```bash
aws application-autoscaling register-scalable-target \
  --service-namespace ecs \
  --scalable-dimension ecs:service:DesiredCount \
  --resource-id service/audio-ai-platform-cluster/user-service \
  --min-capacity 2 \
  --max-capacity 10

aws application-autoscaling put-scaling-policy \
  --service-namespace ecs \
  --scalable-dimension ecs:service:DesiredCount \
  --resource-id service/audio-ai-platform-cluster/user-service \
  --policy-name cpu-scaling-policy \
  --policy-type TargetTrackingScaling \
  --target-tracking-scaling-policy-configuration file://scaling-policy.json
```

#### K8s HPA

```yaml
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: user-service-hpa
  namespace: audio-platform
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: user-service
  minReplicas: 2
  maxReplicas: 10
  metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 70
```

## 12. 安全加固

### 12.1 网络安全

- 使用 VPC 隔离网络
- 配置安全组限制访问
- 使用 WAF 防护 Web 攻击
- 启用 DDoS 防护

### 12.2 数据安全

- 数据库启用加密
- S3 启用服务端加密
- 使用 KMS 管理密钥
- 定期轮换密钥

### 12.3 访问控制

- 使用 IAM 角色管理权限
- 启用 MFA 多因素认证
- 定期审计访问日志
- 最小权限原则
