# 快速开始指南

## 项目概述

Audio AI Platform 是一个基于 AWS 的音频 AI 处理平台，支持：
- IoT 设备音频播放
- 用户上传音频
- 云端 AI 推理
- CDN 分发

## 技术栈

- **后端**: Go 1.22 + go-zero + go-admin
- **数据库**: PostgreSQL + Redis
- **消息队列**: Kafka / SQS
- **云服务**: AWS (S3, ECS, Aurora, ElastiCache)

## 5 分钟快速启动

### 1. 克隆项目

```bash
git clone https://github.com/jacklau/audio-ai-platform.git
cd audio-ai-platform
```

### 2. 🚀 一键启动（推荐）

```bash
# 一键启动所有服务（Docker + go-admin + 微服务）
make start
```

这个命令会自动：
- 启动 Docker 服务（PostgreSQL + Redis）
- 启动 go-admin 后台（端口 8000）
- 启动所有微服务（用户/设备/内容服务）
- 创建日志文件

### 3. 验证服务

```bash
# 用户服务
curl http://localhost:8001/health

# 设备服务
curl http://localhost:8002/health

# 内容服务
curl http://localhost:8003/health

# go-admin 后台
open http://localhost:8000
```

### 4. 停止服务

```bash
# 停止所有服务
make stop
```

## 分步启动（可选）

如果你想分步启动服务：

### 1. 启动 Docker 服务

```bash
make docker-up
```

### 2. 运行数据库迁移

```bash
make migrate-up
```

### 3. 启动 go-admin 后台

```bash
make start-admin
```

### 4. 启动微服务

```bash
make start-services
```

## 项目结构

```

├── services/              # 微服务
│   ├── user/             # 用户服务 (8001)
│   ├── device/           # 设备服务 (8002)
│   ├── content/          # 内容服务 (8003)

├── pkg/                  # 可复用库
├── deploy/               # 部署配置
├── scripts/              # 脚本
└── doc/                  # 文档
```

## 配置说明

### 用户服务配置

```yaml
# services/user/etc/user.yaml
Name: user-api
Host: 0.0.0.0
Port: 8001

DB:
  DataSource: host=localhost port=5432 user=admin password=admin123 dbname=audio_platform sslmode=disable

Redis:
  Host: localhost:6379
  Pass: redis123

Auth:
  AccessSecret: your-secret-key
  AccessExpire: 3600
```

### 环境变量

```bash
# 开发环境
export DATABASE_URL=postgresql://admin:admin123@localhost:5432/audio_platform
export REDIS_ADDR=localhost:6379
export REDIS_PASSWORD=redis123
export AWS_REGION=us-east-1
export AWS_ENDPOINT=http://localhost:4566  # LocalStack
```

## API 测试

### 用户注册

```bash
curl -X POST http://localhost:8001/api/v1/user/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "test@example.com",
    "password": "password123"
  }'
```

### 用户登录

```bash
curl -X POST http://localhost:8001/api/v1/user/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "test@example.com",
    "password": "password123"
  }'
```

### 用户资料

不再提供 `GET /api/v1/user/info`。登录成功响应中含 `user_id`；完整资料可在修改昵称/头像等 `PUT /api/v1/user/info` 的响应中取得，或使用管理端用户详情接口。

## 故障排查

### 服务无法启动

```bash
# 查看日志
docker-compose logs -f

# 检查端口占用
lsof -i :8001
```

### 数据库连接失败

```bash
# 测试连接
psql postgresql://admin:admin123@localhost:5432/audio_platform

# 查看数据库日志
docker logs audio-platform-postgres
```

### Redis 连接失败

```bash
# 测试连接
redis-cli -h localhost -p 6379 -a redis123 ping
```
