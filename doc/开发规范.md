# 开发规范

## 1. 代码规范

### 1.1 命名规范

#### 文件命名
- 使用小写字母和下划线：`user_service.go`
- 测试文件：`user_service_test.go`

#### 变量命名
- 驼峰命名：`userName`, `deviceId`
- 常量使用大写：`const MaxRetry = 3`
- 私有变量小写开头：`userRepo`
- 公开变量大写开头：`UserRepo`

#### 函数命名
- 驼峰命名：`GetUserById`, `createDevice`
- 接口方法大写开头：`type UserService interface { GetUser() }`

### 1.2 注释规范

#### 包注释
```go
// Package user 提供用户相关的业务逻辑
package user
```

#### 函数注释
```go
// GetUserById 根据用户 ID 获取用户信息
// 参数:
//   - id: 用户 ID
// 返回:
//   - *User: 用户对象
//   - error: 错误信息
func GetUserById(id int64) (*User, error) {
    // ...
}
```

#### 复杂逻辑注释
```go
// 计算设备在线率
// 1. 获取所有设备数量
// 2. 获取在线设备数量
// 3. 计算比例
onlineRate := float64(onlineCount) / float64(totalCount)
```

### 1.3 错误处理

#### 错误定义
```go
var (
    ErrUserNotFound = errors.New("user not found")
    ErrInvalidParam = errors.New("invalid parameter")
)
```

#### 错误处理
```go
user, err := GetUserById(id)
if err != nil {
    logger.Error("failed to get user", zap.Int64("id", id), zap.Error(err))
    return nil, err
}
```

#### 错误包装
```go
if err != nil {
    return fmt.Errorf("failed to get user: %w", err)
}
```

### 1.4 日志规范

#### 日志级别
- DEBUG: 调试信息，仅开发环境
- INFO: 一般信息，如请求日志
- WARN: 警告信息，如参数异常
- ERROR: 错误信息，如数据库错误
- FATAL: 致命错误，程序退出

#### 日志格式
```go
logger.Info("user login",
    zap.Int64("user_id", userId),
    zap.String("ip", ip),
    zap.String("device", device),
)

logger.Error("failed to query database",
    zap.String("sql", sql),
    zap.Error(err),
)
```

## 3. API 设计规范

### 3.1 RESTful API

#### 资源命名
- 使用名词复数：`/users`, `/devices`
- 避免动词：`/getUser` ❌ → `/users/{id}` ✅

#### HTTP 方法
- GET: 查询资源
- POST: 创建资源
- PUT: 更新资源（全量）
- PATCH: 更新资源（部分）
- DELETE: 删除资源

#### 示例
```
GET    /api/v1/users          # 获取用户列表
GET    /api/v1/users/{id}     # 获取单个用户
POST   /api/v1/users          # 创建用户
PUT    /api/v1/users/{id}     # 更新用户
DELETE /api/v1/users/{id}     # 删除用户
```

### 3.2 请求参数

#### Query 参数（查询）
```
GET /api/v1/users?page=1&page_size=10&status=active
```

#### Path 参数（资源标识）
```
GET /api/v1/users/{id}
```

#### Body 参数（创建/更新）
```json
POST /api/v1/users
{
  "email": "user@example.com",
  "password": "123456"
}
```

### 3.3 响应格式

#### 成功响应
```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "user_id": 1001,
    "email": "user@example.com"
  }
}
```

#### 错误响应
```json
{
  "code": 1001,
  "msg": "user not found",
  "data": null
}
```

#### 列表响应
```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "list": [...],
    "total": 100,
    "page": 1,
    "page_size": 10
  }
}
```

### 3.4 错误码规范

```
0: 成功
1xxx: 用户相关错误
  1001: 用户不存在
  1002: 密码错误
  1003: 用户已存在
2xxx: 设备相关错误
  2001: 设备不存在
  2002: 设备已绑定
3xxx: 内容相关错误
  3001: 内容不存在
  3002: 上传失败
9xxx: 系统错误
  9001: 数据库错误
  9002: Redis 错误
  9003: 参数错误
```

## 4. 数据库规范

### 4.1 表设计规范

#### 命名规范
- 表名：小写 + 下划线，复数形式：`users`, `devices`
- 字段名：小写 + 下划线：`user_id`, `created_at`
- 索引名：`idx_表名_字段名`：`idx_users_email`
- 外键名：`fk_表名_字段名`：`fk_devices_user_id`

#### 字段规范
- 主键：`id BIGSERIAL PRIMARY KEY`
- 时间戳：`created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP`
- 软删除：`deleted_at TIMESTAMP`
- 状态字段：使用 SMALLINT 或 VARCHAR

#### 索引规范
- 为外键添加索引
- 为常用查询字段添加索引
- 避免过多索引影响写入性能

### 4.2 查询规范

#### 使用参数化查询
```go
// ✅ 正确
db.Query("SELECT * FROM users WHERE id = $1", id)

// ❌ 错误（SQL 注入风险）
db.Query(fmt.Sprintf("SELECT * FROM users WHERE id = %d", id))
```

#### 避免 SELECT *
```go
// ✅ 正确
db.Query("SELECT id, email, nickname FROM users WHERE id = $1", id)

// ❌ 错误
db.Query("SELECT * FROM users WHERE id = $1", id)
```

#### 使用分页
```go
// ✅ 正确
db.Query("SELECT * FROM users LIMIT $1 OFFSET $2", pageSize, offset)
```

### 4.3 事务处理

```go
tx, err := db.Begin()
if err != nil {
    return err
}
defer tx.Rollback()

// 执行多个操作
if err := tx.Exec(...); err != nil {
    return err
}

if err := tx.Exec(...); err != nil {
    return err
}

// 提交事务
return tx.Commit()
```

## 5. Redis 使用规范

### 5.1 Key 命名规范

```
{业务}:{对象}:{id}:{属性}
```

示例：
- `user:session:1001`: 用户会话
- `device:status:2001`: 设备状态
- `content:cache:3001`: 内容缓存

### 5.2 过期时间

- 会话数据：1 小时
- 缓存数据：根据业务设置（5 分钟 - 1 天）
- 临时数据：根据需要设置

### 5.3 数据结构选择

- String: 简单键值对
- Hash: 对象存储
- List: 队列、列表
- Set: 去重集合
- ZSet: 排行榜

## 6. 测试规范

### 6.1 单元测试

```go
func TestGetUserById(t *testing.T) {
    // Arrange
    userId := int64(1001)

    // Act
    user, err := GetUserById(userId)

    // Assert
    assert.NoError(t, err)
    assert.NotNil(t, user)
    assert.Equal(t, userId, user.Id)
}
```

### 6.2 表格驱动测试

```go
func TestValidateEmail(t *testing.T) {
    tests := []struct {
        name    string
        email   string
        wantErr bool
    }{
        {"valid email", "user@example.com", false},
        {"invalid email", "invalid", true},
        {"empty email", "", true},
    }

    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            err := ValidateEmail(tt.email)
            if (err != nil) != tt.wantErr {
                t.Errorf("ValidateEmail() error = %v, wantErr %v", err, tt.wantErr)
            }
        })
    }
}
```

## 9. 性能优化规范

### 9.1 数据库优化

- 使用索引加速查询
- 避免 N+1 查询
- 使用批量操作
- 使用连接池

### 9.2 缓存优化

- 缓存热点数据
- 设置合理的过期时间
- 使用缓存预热
- 防止缓存穿透/击穿/雪崩

### 9.3 代码优化

- 避免不必要的内存分配
- 使用 sync.Pool 复用对象
- 使用 goroutine 并发处理
- 避免阻塞操作

## 10. 安全规范

### 10.1 输入验证

- 验证所有用户输入
- 使用白名单而非黑名单
- 防止 SQL 注入
- 防止 XSS 攻击

### 10.2 认证授权

- 使用 HTTPS
- 密码加密存储
- 使用 JWT 认证
- 实现权限控制

### 10.3 敏感数据

- 不在日志中输出敏感信息
- 敏感数据加密存储
- 使用环境变量管理密钥
- 定期更新密钥

### 11.3 验证流程

#### 标准流程
1. **编写代码**
2. **启动服务** - 验证无报错
3. **测试功能** - 访问接口/查看日志
4. **验证结果** - 确认符合预期
5. **更新文档** - 同步更新相关文档
6. **提交代码** - 附带验证结果

- [ ] 代码能正常编译？
